Dynamic QR Code Generation Process
  1. Faculty Initiates Session: Class start triggers QR window
  2. Server Generates Token: JWT with timestamp + class info
  3. QR Creation: Token encoded in QR with 20-30s expiry
  4. Student Scans: Mobile app decodes and validates
  5. Server Validation: Time window + student eligibility check
  6. Attendance Marking: Record created if validation passes

Technical Implementation Questions

Q: How does your QR code security prevent proxy attendance?

Answer: Our QR system implements multiple security layers:

  • Time-based Expiry: QR codes expire in 20-30 seconds, preventing screenshot sharing
  • JWT Tokens: Each QR contains encrypted payload with timestamp and class information
  • Server-side Validation: Backend validates time window, student enrollment, and prevents duplicate scans
  • Rate Limiting: Maximum one scan per student per class period
  • Class Schedule Verification: System checks if class is actually scheduled at that time

Future Enhancements: GPS validation, device fingerprinting, and facial recognition can add additional security layers.

# Subject Model - Percentage Calculation
class Subject(models.Model):
    total_classes = models.IntegerField(default=0)
    present_classes = models.IntegerField(default=0)
    
    def calculate_percentage(self):
        if self.total_classes > 0:
            self.attendance_percentage = (self.present_classes / self.total_classes) * 100
        else:
            self.attendance_percentage = 0.0
        return self.attendance_percentage
        
    def save(self, *args, **kwargs):
        self.calculate_percentage()  # Auto-calculate on save
        super().save(*args, **kwargs)

# Real-time updates via Django signals
@receiver(post_save, sender=AttendanceRecord)
def update_subject_stats(sender, instance, **kwargs):
    subject = instance.subject
    subject.total_classes = subject.attendance_records.count()
    subject.present_classes = subject.attendance_records.filter(status='P').count()
    subject.save()

Key Points: Automatic calculation, signal-based updates, data consistency maintained.
